          I'm not sure I think real-time validation error-messages really make sense for something like an e-mail address input. You're throwing error-messages at the user while they're still typing... that seems kind of rude and pushy - of course it's not a valid e-mail address, I'm still typing here! ;-)

          A more user-friendly approach, in my opinion, would be to only highlight (color-code) the input - to subtly indicate to the users, while they're still typing, if they're getting it right - but don't display the actual error-messages until they tab out of the input.

          In other words, typing is just normal, natural, expected (unavoidable) interaction with the UI - you can't treat that as an error until the user somehow commits to what they were typing.

          Just my $.02

                                                  Julian Lloyd • 2 years ago

                                                  Thanks for the article!

                                                  With the `ngFocus` example, I had a UX concern with the error message disappearing the instant you re-focus the problematic input—so I modified `ngFocus`, changing it to `ngBlur`

                                                  app.directive('ngBlur', function() {
  return {
    restrict: 'A',
    require: 'ngModel',
    link: function(scope, el, attrs, ctrl) {
      var BLUR_CLASS = "ng-blurred";
      ctrl.$blurred = false;
      el.on('keydown', function(evt) {
        el.removeClass(BLUR_CLASS);
        scope.$apply(function() {
          ctrl.$blurred = false;
        });
      });
      el.on('blur', function(evt) {
        el.addClass(BLUR_CLASS);
        scope.$apply(function() {
          ctrl.$blurred = true;
        });
      });
    }
  };
});

                                                  It's not perfect… the error message incorrectly shows in certain conditions it seems—but I have trouble recreating it consistently. That aside, instead of 

                                                  !signup_form.name.$focused

                                                  You use...

                                                  signup_form.name.$blurred

                                                  and your error messages stay visible until the user modifies the input. Maybe someone can come up with a better way to achieve this?
